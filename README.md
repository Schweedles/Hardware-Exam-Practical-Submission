**Part 3 - ExciteBike Controller** 
**What was done?** 
A custom controller for the NES game _ExciteBike_ was designed. The controller takes the shape of Motorbike handlebars, as seen in the brainstorming/sketch screenshot attached. 
This controller concept was made using Fusion360 and its tools. The aim was to create a controller that from a glance, could be seen as motorbike handlebars, yet still be comfortable to pick up and use similarly to the original NES controller. 

**How was it done? (methods, steps, techniques)**
The creative methods I used when designing this controller were brainstorming and design thinking. Although I was on my own, when design thinking I asked myself how I could see this controller being used and what I needed it to do. From here, I was able to start brainstorming after some initial goals/requirements were met. 

Step 1: Brainstorming
- Asked myself what would be good to have
- outlined requirements for the controller
- looked at different ideas on how to adapt controller

Step 2: Sketching
- Sketched a quick concept of what I thought could work

Step 3: CAD 
- Went into Fusion and started modelling handle pieces and controller body
- Created controller body, then had the handle bars come out of the middle piece. This was done becuase of the potential complexity when joining separate handlebar pieces to the controller's main body. 
- After all modeling was done, created technical drawing and Bill of Materials. 

Step 4: Write up/explanation. 
- Wrote in this ReadMe about the process and steps. 
The first step I took when designing this controller was the ideation and brainstorming.

**Why was it done this way?**
The process: 
Even though it was a short time frame, I found it useful to go over a quick brainstorming process of asking myself what the requirements should be. How many buttons it should have, how big it should be, and what the form factor should be. This let me streamline my design process a little bit more and gave me direction when starting. 

The Design: 
The reason the controller is shaped this was is to give it the user a pretty good idea of how to hold it and what it is for. The handlebar shape is semi-recognizable and it is relatively clear where the user should place their hands/thumbs to access the buttons. This increases immersion while retaining the knowledge of how the controller could/should be used.
